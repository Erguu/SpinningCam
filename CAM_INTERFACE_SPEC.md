# CAM → PLC Interface Specification
## Mexico Metal Spinning — S7-1200 Recipe Format v2.0

---

## Purpose of This Document

This document tells the CAM program (or the AI generating it) exactly what output format the PLC expects. Read this document completely before generating any output. The PLC will reject or misexecute any program that does not follow this spec.

---

## 1. Machine Description

**Type:** 2-axis CNC metal spinning lathe
**Controller:** Siemens S7-1200 (CPU 1214C)
**Motion axes:** X (radial) + Z (axial) — simultaneous linear interpolation
**Spindle:** Rotates the workpiece blank on the mandrel (not a cutting spindle)
**Tool:** A forming roller mounted on a 4-position turret, moved by the CNC axes

**What the machine does:**
The blank (sheet metal disc) is clamped against a rotating mandrel. The CNC-controlled roller presses and traces along the mandrel profile in multiple passes, progressively forming the blank into a 3D shape.

---

## 2. Coordinate System

```
                SPINDLE AXIS (Z)
                ──────────────────────────────► Z positive
                │
                │  Z = 0: mandrel face / home position
                │  Z = 200: maximum axial reach
                │
  X = 0         │
  (center)      │
      ──────────┤
                │
                │  X = 0: spindle center axis (theoretical)
                ▼  X = 170: maximum radial reach
                X positive (radial)
```

**Rules that must never be violated:**

| Axis | Home (origin) | Direction | Soft limit min | Soft limit max |
|------|--------------|-----------|----------------|----------------|
| X    | 0.0 mm       | Radially outward from spindle center | 0.0 mm | 170.0 mm |
| Z    | 0.0 mm       | Away from mandrel face (toward operator) | 0.0 mm | 200.0 mm |

- **X and Z values must always be ≥ 0.** There are no negative coordinates.
- Both axes start at home (0, 0) after homing.
- The PLC pre-scans all lines before execution and rejects the program if any coordinate is outside the soft limits.

## 3. Output Format

The CAM program must output a single `.scl` file containing one `DATA_BLOCK` per program. This file is imported directly into TIA Portal.

### 3.1 File Template

```scl
// ============================================
// DB_RecipeProgram[N] — [Descriptive Program Name]
// Lines: [LineCount]
// Part: [Part Name or Description]
// Generated by: [CAM Program Name + Version]
// ============================================

DATA_BLOCK "DB_RecipeProgram[N]"
{ S7_Optimized_Access := 'TRUE' }
VERSION : 0.1
NON_RETAIN
    VAR
        Header : "RecipeHeader";
        Lines : Array[0..999] of "RecipeLine";
    END_VAR
BEGIN
    // --- Header ---
    Header.sName := '[Program Name max 20 chars]';
    Header.LineCount := [N];
    Header.Valid := TRUE;

    // --- Lines ---
    Lines[0].X := 0.000; Lines[0].Z := 0.000; Lines[0].F := 0; Lines[0].CMD := 20; Lines[0].Param := 100; // SPINDLE_ON 1000 RPM
    Lines[1].X := 10.000; Lines[1].Z := 5.000; Lines[1].F := 0; Lines[1].CMD := 0; Lines[1].Param := 0;   // RAPID
    Lines[2].X := 50.000; Lines[2].Z := 50.000; Lines[2].F := 300; Lines[2].CMD := 1; Lines[2].Param := 0; // LINEAR
    // ...
    Lines[N-2].X := 10.000; Lines[N-2].Z := 5.000; Lines[N-2].F := 0; Lines[N-2].CMD := 21; Lines[N-2].Param := 0; // SPINDLE_OFF
    Lines[N-1].X := 0.000; Lines[N-1].Z := 0.000; Lines[N-1].F := 0; Lines[N-1].CMD := 99; Lines[N-1].Param := 0;  // PROGRAM_END
END_DATA_BLOCK
```

Replace `[N]` with 1–5 (program slot number).

## 4. RecipeLine Fields

Each line is exactly 12 bytes:

| Field | SCL Type | Bytes | Description |
|-------|----------|-------|-------------|
| X | Real | 4 | Target X position (mm). Must be 0.0 – 170.0. |
| Z | Real | 4 | Target Z position (mm). Must be 0.0 – 200.0. |
| F | Int  | 2 | Feedrate in **mm/min**, integer. Used only for CMD=1 (LINEAR). Set 0 for all other CMDs. |
| CMD | Byte | 1 | Command type — see table in Section 5. |
| Param | Byte | 1 | Parameter — meaning depends on CMD, see Section 5. |

**Important:** X and Z are always the **target** position, not a delta/increment.
**Important:** F is an integer (whole number). Round feedrate to the nearest integer.

---

## 5. Command (CMD) Reference

| CMD | Name | X,Z used? | F used? | Param meaning | Equivalent G-code |
|-----|------|-----------|---------|---------------|-------------------|
| 0   | RAPID | Yes | No (ignored) | 0 | G0 X Z |
| 1   | LINEAR | Yes | Yes | 0 | G1 X Z F |
| 10  | TOOL_CHANGE | No (ignored) | No | External tool code (0–255) | M6 T |
| 20  | SPINDLE_ON | No | No | Speed = Param × 10 RPM | M3 S |
| 21  | SPINDLE_OFF | No | No | 0 | M5 |
| 30  | DWELL | No | No | Time = Param × 100 ms | G4 P |
| 40  | CYLINDER_GOTO | No | No | Position = Param × 10 mm | M40 P |
| 99  | PROGRAM_END | No | No | 0 | M30 |

### RAPID (CMD=0)
- Both axes move simultaneously at maximum rapid speed (50 mm/s = 3000 mm/min).
- FeedrateOverride from HMI also applies to rapid (50–200%).
- Use for approach, retract, and reposition moves — not for forming contact.

### LINEAR (CMD=1)
- Both axes move simultaneously with linear interpolation.
- PLC calculates per-axis velocity from total path length and feedrate.
- **F field:** feedrate in mm/min, integer, range 1–3000. Values above 3000 are clamped to 3000 by the PLC.
- The HMI FeedrateOverride (50–200%) is applied at runtime, so program with nominal feed.

### TOOL_CHANGE (CMD=10)
- Param carries the **external tool code** — not the physical turret slot number.
- External codes are arbitrary numbers (0–255) the operator assigns to physical slots on the HMI Tool Setup screen.
- The CAM program and operator must agree on which code means which tool.
- **If the code is not found in the mapping table, the program halts with an error.** The operator must open Tool Setup, enter the code for the correct slot, and restart.
- The PLC automatically moves the turret; the CAM does not need to add positioning moves for the turret rotation.
- **Add retract moves (RAPID to safe position) before and after TOOL_CHANGE** — the turret rotates only, axes do not move automatically.

### SPINDLE_ON (CMD=20)
- Param encodes speed: `Param = round(RPM / 10)`
- Speed range via Param: 100 RPM (Param=10) to 2550 RPM (Param=255)
- Note: DB_Spindle.MaxSpeed = 3000 RPM but Param max (255) = 2550 RPM. For higher speeds, a future protocol update is needed.
- Always emit SPINDLE_ON before any forming move.
- SPINDLE_ON with a new Param while spindle is running changes the speed target.

| Desired RPM | Param value |
|-------------|-------------|
| 500 RPM     | 50          |
| 800 RPM     | 80          |
| 1000 RPM    | 100         |
| 1500 RPM    | 150         |
| 2000 RPM    | 200         |
| 2550 RPM    | 255 (max)   |

### SPINDLE_OFF (CMD=21)
- Always emit before PROGRAM_END.
- Emit before TOOL_CHANGE if changing at low or zero speed.

### DWELL (CMD=30)
- Pause for `Param × 100 ms`.
- Example: Param=10 → 1 second pause.
- Max dwell: 255 × 100 ms = 25.5 seconds.

### PROGRAM_END (CMD=99)
- Must be the last line of every program.
- After PROGRAM_END the PLC moves to COMPLETE state and stops all motion.
- A program without PROGRAM_END will never cleanly terminate.

## 7. Feedrate Guidelines

| Move type | Typical F (mm/min) | Notes |
|-----------|-------------------|-------|
| Rapid (G0) | — | Always CMD=0, F not used |
| Light first pass | 100–200 | Initial forming contact |
| Normal forming pass | 200–600 | Typical spinning pass |
| Finishing pass | 100–300 | Reduced for surface quality |
| Deep push / aggressive | up to 1000 | Heavy forming, thick blanks |
| Maximum (hard cap) | 3000 | PLC clamps everything above this |

## 8. Required Program Structure

Every program must follow this skeleton:

```
1. SPINDLE_ON (CMD=20)         — start spindle FIRST
2. RAPID (CMD=0) to safe start — approach from home safely
3. ... forming passes ...
4. RAPID (CMD=0) to retract    — clear before tool change (if any)
5. TOOL_CHANGE (CMD=10)        — optional; only if tool change needed
6. RAPID (CMD=0) to safe start — re-approach after tool change
7. ... more forming passes ...
8. RAPID (CMD=0) to home area  — return axes to safe zone
9. SPINDLE_OFF (CMD=21)        — stop spindle
10. PROGRAM_END (CMD=99)       — terminate program
```

**Rules:**
- The very first line must be SPINDLE_ON (or a RAPID to a safe position if spindle start is deferred — not recommended).
- The last two lines must always be SPINDLE_OFF then PROGRAM_END.
- Do not move axes while spindle is off (except repositioning before start or after stop).
- Always include a RAPID retract to a safe X,Z before any TOOL_CHANGE.
- Never exceed 1000 lines total per program.

## 10. Required PLC Data Types

These types must be defined in `01_DataTypes.scl` in TIA Portal before importing any `DB_RecipeProgram`:

```scl
TYPE "RecipeLine"
VERSION : 0.1
    STRUCT
        X : Real;       // Target X position (mm)
        Z : Real;       // Target Z position (mm)
        F : Int;        // Feedrate (mm/min) — integer
        CMD : Byte;     // Command type (see Section 5)
        Param : Byte;   // Command parameter (meaning depends on CMD)
    END_STRUCT;
END_TYPE

TYPE "RecipeHeader"
VERSION : 0.1
    STRUCT
        sName : String[20];   // Program name — note: field is sName, not Name
        LineCount : Int;      // Number of initialized lines (Lines[0] to Lines[LineCount-1])
        Valid : Bool;         // TRUE = program is ready to execute
        PreScanned : Bool;    // Set to TRUE by PLC after pre-scan passes
        MinX : Real;          // Bounding box min X (filled by CAM, used for pre-scan)
        MaxX : Real;          // Bounding box max X
        MinZ : Real;          // Bounding box min Z
        MaxZ : Real;          // Bounding box max Z
    END_STRUCT;
END_TYPE
```

**Important:** The header field is `sName` (not `Name`). This matches the SCL generated by `recipe_to_scl.py`.

## 11. Complete Example

A simple bowl-forming program (illustrative coordinates):

```scl
DATA_BLOCK "DB_RecipeProgram1"
{ S7_Optimized_Access := 'TRUE' }
VERSION : 0.1
NON_RETAIN
    VAR
        Header : "RecipeHeader";
        Lines : Array[0..999] of "RecipeLine";
    END_VAR
BEGIN
    Header.sName := 'Bowl_Pass3';
    Header.LineCount := 14;
    Header.Valid := TRUE;

    // Line 0: Start spindle at 1200 RPM
    Lines[0].X := 0.000; Lines[0].Z := 0.000; Lines[0].F := 0; Lines[0].CMD := 20; Lines[0].Param := 120;

    // Line 1: Rapid to safe start position
    Lines[1].X := 10.000; Lines[1].Z := 5.000; Lines[1].F := 0; Lines[1].CMD := 0; Lines[1].Param := 0;

    // Line 2: Rapid approach to first contact point
    Lines[2].X := 30.000; Lines[2].Z := 80.000; Lines[2].F := 0; Lines[2].CMD := 0; Lines[2].Param := 0;

    // Line 3–9: Forming passes (linear moves with feedrate)
    Lines[3].X := 40.000; Lines[3].Z := 75.000; Lines[3].F := 300; Lines[3].CMD := 1; Lines[3].Param := 0;
    Lines[4].X := 60.000; Lines[4].Z := 60.000; Lines[4].F := 300; Lines[4].CMD := 1; Lines[4].Param := 0;
    Lines[5].X := 80.000; Lines[5].Z := 45.000; Lines[5].F := 300; Lines[5].CMD := 1; Lines[5].Param := 0;
    Lines[6].X := 100.000; Lines[6].Z := 30.000; Lines[6].F := 250; Lines[6].CMD := 1; Lines[6].Param := 0;
    Lines[7].X := 120.000; Lines[7].Z := 18.000; Lines[7].F := 250; Lines[7].CMD := 1; Lines[7].Param := 0;
    Lines[8].X := 140.000; Lines[8].Z := 8.000; Lines[8].F := 200; Lines[8].CMD := 1; Lines[8].Param := 0;
    Lines[9].X := 155.000; Lines[9].Z := 2.000; Lines[9].F := 150; Lines[9].CMD := 1; Lines[9].Param := 0;

    // Line 10: Retract radially
    Lines[10].X := 10.000; Lines[10].Z := 5.000; Lines[10].F := 0; Lines[10].CMD := 0; Lines[10].Param := 0;

    // Line 11: Return to home area
    Lines[11].X := 5.000; Lines[11].Z := 5.000; Lines[11].F := 0; Lines[11].CMD := 0; Lines[11].Param := 0;

    // Line 12: Spindle off
    Lines[12].X := 0.000; Lines[12].Z := 0.000; Lines[12].F := 0; Lines[12].CMD := 21; Lines[12].Param := 0;

    // Line 13: Program end
    Lines[13].X := 0.000; Lines[13].Z := 0.000; Lines[13].F := 0; Lines[13].CMD := 99; Lines[13].Param := 0;

END_DATA_BLOCK
```

## 11. Common Mistakes to Avoid

| Mistake | Effect | Correct approach |
|---------|--------|-----------------|
| Negative X or Z coordinate | Pre-scan rejects the program | All coordinates ≥ 0.0 |
| X > 170 or Z > 200 | Pre-scan rejects the program | Verify all coordinates against limits |
| F value not integer (e.g. 312.5) | SCL compile error — Int type | Round to nearest integer: 313 |
| F > 3000 | PLC silently clamps to 3000 mm/min | Stay within 1–3000 range |
| Missing PROGRAM_END | Program never reaches COMPLETE state | Always end with CMD=99 |
| SPINDLE_OFF missing before END | Spindle keeps running after program ends | Always emit CMD=21 before CMD=99 |
| Tool code not in mapping table | Program halts with error 0x0308 | Coordinate codes with operator before running |
| Axes moving before SPINDLE_ON | Potential for unintended contact | Always start spindle first |
| X=0 during forming | Risk of collision with spindle | Ensure X ≥ 5.0 mm during any forming move |
| Header.LineCount wrong | PLC stops at wrong line or runs garbage | LineCount = exact number of initialized lines |
| Lines[] index gap | PLC reads uninitialized line | Lines must be contiguous from 0 to LineCount-1 |

## 12. Tool Change Workflow (for CAM programmer)

1. **Decide on external codes** together with the machine operator before writing any program. Example: code 11 = roughing roller, code 12 = finishing roller.
2. In the CAM program, emit `TOOL_CHANGE` (CMD=10) with `Param = chosen code`.
3. The operator enters the same codes into the HMI Tool Setup screen, assigning each code to a physical turret slot (1–4).
4. If the operator changes which physical tool is in a slot, they update the HMI — the CAM program does not change.

**CAM output for a tool change:**
```scl
// Retract before tool change
Lines[i].X := 10.000; Lines[i].Z := 10.000; Lines[i].F := 0; Lines[i].CMD := 0; Lines[i].Param := 0; // RAPID safe pos
// Tool change — external code 12 (finishing roller)
Lines[i+1].X := 0.000; Lines[i+1].Z := 0.000; Lines[i+1].F := 0; Lines[i+1].CMD := 10; Lines[i+1].Param := 12;
// Approach again after tool change
Lines[i+2].X := 30.000; Lines[i+2].Z := 80.000; Lines[i+2].F := 0; Lines[i+2].CMD := 0; Lines[i+2].Param := 0; // RAPID
```

## 13. Validation Checklist (run before output)

Before finalizing the SCL file, verify:

- [ ] All X values: 0.0 ≤ X ≤ 170.0
- [ ] All Z values: 0.0 ≤ Z ≤ 200.0
- [ ] All F values for CMD=1: integer, 1–3000
- [ ] F = 0 for CMD = 0, 10, 20, 21, 30, 99
- [ ] Param within Byte range (0–255) for all lines
- [ ] First line is SPINDLE_ON (CMD=20) or safe RAPID
- [ ] Second-to-last line is SPINDLE_OFF (CMD=21)
- [ ] Last line is PROGRAM_END (CMD=99)
- [ ] Header.LineCount matches actual number of initialized Lines[]
- [ ] Lines are contiguous from Lines[0] to Lines[LineCount-1]
- [ ] Total lines ≤ 1000
- [ ] Header.sName ≤ 20 characters
- [ ] DB name matches slot: `DB_RecipeProgram[1-5]`
- [ ] File includes `NON_RETAIN` keyword
